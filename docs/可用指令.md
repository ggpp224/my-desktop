## 一、工作流

| 指令示例 | 说明 |
|----------|------|
| **开始工作** | 执行工作流 `start-work`：依次启动 cpxy、react18、base18、cc-web、biz-solution、uikit、shared，并执行 docker compose。 |
| 执行工作流 start-work | 同上。 |
| 执行工作流 standalone | 执行 `standalone` 工作流（如仅启动 scm）。 |
| **升级集测react18的nova版本** | 执行 `upgrade-react18-nova` 工作流：自动 fetch、切到最大 sprint 分支、拉取代码、解析最新 `1.xxxxxx.y` 的 `@chanjet/nova-uikit` 版本、更新指定依赖、提交、push，并切回原分支。 |
| **升级集测cc-web的nova版本** | 执行 `upgrade-cc-web-nova` 工作流：自动 fetch、切到最大 sprint 分支、拉取代码、解析最新 `1.xxxxxx.y` 的 `@chanjet/nova-uikit` 版本、更新指定依赖、提交、push，并切回原分支。 |

**单独执行工作流中的某一步：**

| 指令示例 | 说明 |
|----------|------|
| 启动 cpxy | 仅执行 start-work 中 taskKey 为 cpxy 的步骤。 |
| 启动 react18 / 启动 base18 / 启动 cc-web / 启动 biz-solution | 在对应项目目录下执行命令（如 yarn dev）。 |
| 启动 uikit / 启动 shared | 同上，针对 nova-next 子包。 |
| 启动 scm | 若用 standalone 工作流则启动 scm；若用 start-work 则执行 start-work 中 taskKey 为 scm 的步骤。 |

工作流步骤的 `taskKey` 与项目代号由 `workflows/*.json` 和 `config/projects` 决定。

## 二、Jenkins 部署

| 指令示例 | 说明 |
|----------|------|
| **部署 nova** / 帮我部署 cc-web / 部署 base | 触发对应 Jenkins 任务，使用预定义分支（nova 用 test；base、base18、cc-web、scm、react18、biz-solution、biz-guide 用 test-260127）。 |
| 部署 react18 / 部署 biz-solution / 部署 biz-guide / 部署 scm / 部署 base18 | 同上，job 为预定义代号。 |
| **部署 nova 分支是 sprint-260326** / 部署 cc-web 分支是 xxx | 指定分支部署：branch 会作为 Jenkins 参数 BRANCH_NAME 传入，不指定则用该项目默认分支。 |

**支持的部署代号：** nova、cc-web、cc-node、react18、base、base18、biz-solution、biz-guide、scm。也可传入完整 Jenkins Job 名称。  
若 Jenkins 返回 404，请确认 Jenkins 上存在该任务，且任务名与 `config/projects.ts` 中对应代号的 `jenkins.jobName` 一致。

## 四、用 IDE 打开 / 关闭项目

项目代号与本地路径在 `config/projects` 中配置，完整列表见下方 **六、项目代号速查**。

### 4.1 用指定应用打开项目

| 指令示例 | 说明 |
|----------|------|
| **ws打开base** / 用 WebStorm 打开 base | 用 WebStorm 打开 base 对应目录。 |
| **cursor打开base** / 用 Cursor 打开 scm | 用 Cursor 打开对应项目目录。 |
| ws打开nova / cursor打开react18 / 用 VS Code 打开 cc-web | 同上，app 与项目代号组合。 |

**应用别名：** `ws` 或 `webstorm` → WebStorm；`cursor` → Cursor；`vscode` 或 `code` → Visual Studio Code。  
当前仅 macOS 支持（使用 `open -a`）。

### 4.2 关闭 IDE 中已打开的项目窗口

| 指令示例 | 说明 |
|----------|------|
| **关闭ws的nova** / 关闭 WebStorm 的 nova | 在 WebStorm 中关闭标题包含该项目目录名的窗口（通过菜单 File → Close Project）。 |
| **关闭cursor的base** / 关闭 Cursor 的 scm | 在 Cursor 中关闭对应项目窗口（Cmd+W）。 |

WebStorm/JetBrains 通过菜单「关闭项目」；Cursor/VS Code 通过 Cmd+W。当前仅 macOS 支持。
