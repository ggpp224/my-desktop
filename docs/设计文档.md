## 1. 系统架构

### 1.1 总体架构

```
用户
  ↓
Desktop UI (Electron + React)
  ↓
Node 后端 (Express)
  ↓
Agent 模块
  ↓
本地 LLM (Ollama / Qwen2.5)
  ↓
Tool Router
  ↓
Tools (Shell / Browser / Jenkins / Workflow)
```

- **Desktop UI**：Electron 承载 React 应用，开发时加载 Vite dev server，生产时加载打包后的静态资源。
- **后端**：Express 提供 HTTP API；生产环境下由 Electron 主进程启动，开发时可单独进程。
- **Agent**：接收用户消息，调用 Ollama 的 chat + tools 接口，解析 `tool_calls`，经 Tool Router 执行工具，将结果回传 LLM 或直接返回前端。
- **Tools**：统一由 Tool Router 按名称分发，各自无状态、可独立测试。

### 1.2 数据流

1. 用户在前端输入或点击快捷按钮 → `POST /agent/chat`，body `{ "message": "..." }`。
2. Agent 将 message 与 tools schema 发给 Ollama `/api/chat`。
3. Ollama 返回 `message.tool_calls`（或纯文本）；Agent 解析为 `{ name, arguments }` 列表。
4. Tool Router 根据 `name` 调用对应 tool（shell/browser/jenkins/workflow），传入 `arguments`。
5. 工具执行结果写回 messages，再次调用 Ollama 生成最终回复（或直接返回）。
6. API 返回 `{ success, text?, toolResults?, error? }`，前端展示在 Chat 与 Logs 中。

## 2. 模块设计

### 2.1 Agent 层（agent/）

| 文件 | 职责 |
|------|------|
| `agent.ts` | 对外 `runAgent(message)`：组 messages、调 Ollama、解析 tool_calls、循环执行并回传结果，返回统一结果结构。 |
| `ollama-client.ts` | 封装 `chatWithTools(messages, tools)`、`healthCheck()`、`parseToolCalls(message)`，与 Ollama API 对接。 |
| `tool-router.ts` | `routeAndExecute(call)`：根据 `call.name` 分发到 shell/browser/jenkins/workflow，返回执行结果。 |
| `tools-schema.ts` | 定义传给 Ollama 的 tools JSON schema（name、description、parameters），与各 tool 实现一一对应。 |

### 2.2 Tools 层（tools/）

| 工具 | 接口 | 说明 |
|------|------|------|
| shell-tool | `run(command, options?)` | `child_process.exec`，cwd 来自 config，支持高危命令检测与二次确认占位。 |
| browser-tool | `open(url)` | 按平台使用 `open` / `start` / `xdg-open` 打开 URL。 |
| jenkins-tool | `deploy(jobName)` | `POST {JENKINS_BASE_URL}/job/{jobName}/build`，可选 Bearer token。 |
| workflow-tool | `runWorkflow(name)` | 从 `workflows/{name}.json` 读取 steps，顺序执行；某步失败即中止并返回错误。 |

### 2.3 工作流格式（workflows/*.json）

```json
{
  "steps": [
    { "tool": "shell", "cmd": "docker compose up -d" },
    { "tool": "browser", "url": "https://example.com" },
    { "tool": "jenkins", "jobName": "my-job" }
  ]
}
```

- 仅支持 `shell` / `browser` / `jenkins` 三种 step；顺序执行，单步失败即中止。

### 2.4 后端 API（server/api.ts）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 调用 Ollama 健康检查，返回 `{ ok, service }`，用于前端提示「请先启动 Ollama」。 |
| POST | `/agent/chat` | Body `{ "message": "..." }`，返回 `runAgent` 结果：`{ success, text?, toolResults?, error? }`。 |

- 使用 CORS、express.json()；敏感配置不落库，从环境变量读取。

### 2.5 配置（config/default.ts）

- **ollama**：`baseUrl`、`model`（环境变量 `OLLAMA_BASE`、`OLLAMA_MODEL`）。
- **server**：`port`（`PORT`）。
- **jenkins**：`baseUrl`、`token`（`JENKINS_BASE_URL`、`JENKINS_TOKEN`）。
- **shell**：`allowedCwd`（`SHELL_CWD`，默认 `process.cwd()`）。

## 3. 前端设计

### 3.1 技术栈

- React 18 + Vite + TypeScript；构建产物由 Electron 加载（生产时 `loadFile(ui/dist/index.html)`）。

### 3.2 页面结构

- **Header**：标题 + Ollama 未就绪时的提示（依赖 `/health`）。
- **左侧边栏**：Workflow Panel（工作流列表与运行）+ Tool Panel（工具列表）。
- **主区域**：Chat Panel（快捷按钮、消息列表、输入框）+ 下方 Logs Panel。

### 3.3 与后端交互

- 基础 URL：`VITE_API_URL` 或 `http://localhost:3000`。
- 启动时请求 `GET /health` 决定是否展示 Ollama 提示。
- 发送指令：`POST /agent/chat`，展示返回的 `text` 与 `toolResults`，并在 Logs 中追加记录。

## 4. 安全与错误处理

- **Shell**：仅在配置的 `allowedCwd` 下执行；高危命令可通过 `requireConfirmation` 或后续白名单/黑名单限制。
- **敏感信息**：Jenkins Token、Ollama 等仅通过环境变量或本地配置注入，不写入代码与仓库。
- **Ollama 不可用**：`/health` 返回 503 或失败时，前端展示「请先安装并启动 Ollama」及文档链接。
- **LLM 无 tool call**：Agent 将纯文本回复或「未解析到可执行操作」返回前端，不抛未捕获异常。
- **Workflow 单步失败**：当前策略为中止并返回错误信息，后续可扩展「跳过/重试」配置。

## 5. 项目目录与构建

- 根目录即项目根（my-desktop），包含 `agent/`、`tools/`、`workflows/`、`server/`、`desktop/`、`config/`、`ui/`、`scripts/`、`docs/`。
- 构建：`tsc` 输出到 `dist/`（不含 ui）；`ui` 使用 Vite 构建到 `ui/dist`。Electron 入口为 `dist/desktop/electron-main.js`，生产时主进程在该入口中启动 Express 并加载 `ui/dist`。
